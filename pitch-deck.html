03 / 08
How it works

A grounded observer. A natural voice.

The hard part isn't talking — it's not hallucinating. So the agent has two brains: one writes down only what's literally in the frame; the other speaks like a friend watching over your shoulder.

STEP 01 · PERCEPTION

Cold observer writes the facts.

Gemini 2.5 Flash returns strict JSON per frame — objects, device_state, visible_features, changed_vs_prior. Ground truth only, no user-action inference.

STEP 02 · GROUNDING

Voice model narrates against those facts.

Gemini Live sees the same frame with its own eyes but must use the JSON as a guardrail. It can't contradict it. Natural delivery, no hallucinations.

STEP 03 · COACHING

Manuals and web search on demand.

Tools load mid-conversation — screw counts, hidden fasteners, model-specific warnings. The agent doesn't advance until the camera confirms the step.

04 / 08
Tools

It calls tools when the answer lives outside its head.

The agent isn't just a camera and a voice. When a brand name appears or a model number is asked, it reaches for the right source — without breaking flow.

lookup_manual

Internal · RAG

A library of repair playbooks indexed per device. Pulls exact procedures — screws, tools, hidden fasteners — and narrates them step by step.

Call
tool_call(lookup_manual)
  query: "Morphy Richards trimmer disassembly"
Result
total_screws: 3  # 1 back + 2 hidden
tool: "H1 hex"
warnings: ["battery still live"]
Agent speaks
"Three H1 screws — one at the back,
 two hidden under the front hood.
 Battery stays live, don't short it."

web_search

Live · Google

When the manual doesn't cover it — rare models, error codes, recalls — the agent searches the live web, summarizes, and speaks the fix.

Call
tool_call(web_search)
  query: "DeLonghi EC155 E3 error reset"
Result
source: "delonghi.com/support"
summary: "E3 = pump priming fault.
 Hold STEAM + COFFEE 5 sec."
Agent speaks
"E3 means the pump didn't prime.
 Hold STEAM + COFFEE five seconds,
 then run a water cycle."
05 / 08
Offline mode

Breakdown in a dead zone? It still works.

Car stalls on a no-signal stretch of highway. Factory's air-gapped. Offshore rig. Hospital that blocks outbound traffic. The two-stage architecture is model-agnostic — swap the cloud backbone for on-device models and zero bytes leave the device.

Cloud mode

Default

Best voice, richest reasoning, live tool calls. Default path when the network is strong.

Stack
voice:       Gemini Live · native audio
perception:  Gemini 2.5 Flash · JSON
tools:       lookup_manual · web_search
latency:     ~300 ms to first word

On-device fallback

Offline · 0 bytes

A laptop, phone, or Pi-class board runs the whole loop locally. Same prompts, same tool schemas, same UI — only the model backbone swaps.

Stack
voice in:    whisper.cpp · ggml
perception:  Moondream / Qwen2-VL
voice out:   Piper · neural TTS
manuals:     local vector store
