How it works

A grounded observer. A natural voice. One agent.

The hard part isn't talking — it's not hallucinating. So the agent has two brains: one that writes down only what it literally sees, and one that speaks like a friend watching over your shoulder.

STEP 01 · PERCEPTION

A cold observer writes down the facts.

Every frame is checked by a separate perception pass that emits strict JSON — objects, device state, visible features, what changed. No inference about user actions. Ground truth only.

STEP 02 · GROUNDING

The voice model narrates against those facts.

A realtime voice model sees the same frame with its own eyes, but must use the JSON as a guardrail. It can't contradict it. That's how you keep natural delivery without making things up.

STEP 03 · COACHING

Manuals and web search arrive on demand.

When a device appears, the agent pulls the repair manual or searches the web — screw counts, hidden fasteners, model-specific warnings — and walks each step. It doesn't advance until the camera confirms the step is done.

Tools

It calls tools when the answer lives outside its head.

The agent isn't just a camera and a voice. When a brand name appears or a model number is asked, it reaches for the right source — an internal repair manual or a web search — mid-conversation, without breaking flow.

lookup_manual

Internal · RAG

A library of repair playbooks indexed per device. The agent pulls the exact procedure — screws, tools, hidden fasteners — and narrates it step by step.

Under the hood

The hard engineering is in the silence.

A voice model that talks nonstop is noise. One that waits for the right moment, and never says a word it can't ground in a pixel, is a coach.

Two-stage
Perception separated from speech.

Flash writes the facts. Live reads them and speaks. Neither model is asked to do both jobs at once — that's the trick.

Event-driven
Frame-diff triggers, not fixed timers.

We don't nudge on a clock. We nudge when the pixels actually change — so the agent speaks when something happens, not when a timer fires.

Non-blocking
Perception runs in parallel.

Flash calls are fired as background tasks so the WebSocket frame loop never stalls — no backpressure, no disconnects, smooth audio.

Anti-hallucination
Forbidden-phrase guardrails.

The agent never confirms a claim from the user's words alone. "I removed the screw" doesn't become "I can see the screw is out" — only the camera decides.
